The Most Desirable Data Scientist in the U.S.

Posted on Tue 13 October 2015 in Python

Earlier this year I applied to the Recurse Center (RC), excited about the opportunity to learn and practice some of the skills that frequently show up on "Data Scientist" job postings:

Dedicating three months to exploring new programming things and connecting with real data scientists in the wild seemed like a smart and fun move. When I got here, I struggled to pick a project during the first week but eventually settled on building a python webscraper to collect data from "Data Scientist" job postings on Indeed.com. Later, I extended it to collect data from LinkedIn postings as well, for reasons I'll elaborate on shortly. In addition to being a fun first project, I can now confidently include a few of the most-desired skills on my resume!

Although a CSV would have sufficed for the scale of this project, I stored the data in a sqlite database because that's one more wrench I wanted to add to my data science toolbox. The schema consists of three tables: (i) posts, which includes data about individual job postings such as company and location, (ii) skills, which is just a list of data science skills, each with a unique id (I curated a dictionary of data science skills by manually adding and deleting things from a starter list of programming languages that I scraped from Wikipedia.), and (iii) post_skills, which keeps track of the many-to-many relationship between job posts and skills by storing rows of the form [job post id, skill id]. Each job post can contain zero or more desired skills, and conversely, each skill in the skills table is associated with one or more job posts (I did not include skills listed in my dictionary in the sqlite skills table unless it was found in at least one job post).

Let's start by counting the number of job posts that requested each skill.

# The pandas library is used for creating and manipulating data frames in python
import pandas as pd
# The sqlite3 library is used to set up a local database that doesn't require a server
import sqlite3 as lite

# Connect to LinkedIn database
lCon = lite.connect('linkedin.db')
lCur = lCon.cursor()

# Connect to Indeed database
iCon = lite.connect('indeed.db')
iCur = iCon.cursor()

# This query will count the number of rows associated with each skillId and order them (increasing)
# The nested select statement (in parentheses) gets the skill names associated with the counts
query = "select (select skill from skills where id=skillId), count(*) from post_skills group by skillId order by count(*)"
lCounts = lCur.execute(query).fetchall()
iCounts = iCur.execute(query).fetchall()
lResults = pd.DataFrame(lCounts, columns=["skill", "count"])
iResults = pd.DataFrame(iCounts, columns=["skill", "count"])

# Get the total number of posts in each database to convert counts to proportions
query = "select count(*) from posts"
lN = lCur.execute(query).fetchall()[0][0]
iN = iCur.execute(query).fetchall()[0][0]
lCon.close()
iCon.close()
lResults['proportion'] = pd.Series(lResults['count'])/lN
iResults['proportion'] = pd.Series(iResults['count'])/iN

I originally built the webscraper for grabbing data from Indeed.com. The biggest challenge was that most of the links on Indeed redirect to the company that is advertising the job. This means I couldn't work with any standard HTML layout or CSS selectors and instead had to parse the text using regular expressions (yet another newish technique to me that I wanted to learn more about here at RC). For example, some sites list desired skills in bulleted lists while others put them right in the text paragraphs, with variations on form, including: (i) SQL/Python/R/SAS/C++, (ii) SQL, Python, R, SAS, C++, (iii) (SQL, Python, R, SAS), and more!

Of course, the most difficult skill to parse was R. I was worried my parser was picking up some garbage that was inflating R's count. Indeed (ha!), the beta version of my parser was picking up a few R imposters, so I had to improve my regex to deal with the fakers. It's possible a few still slipped by, but comparing the results to the LinkedIn scraper, the results seem about right. (This is before I discovered the best regex trick ever!) The LinkedIn jobs were much easier to parse because they all have a standard "job description" section where skills are listed. Thus, the fact that R appeared in approximately 50% of job posts in both datasets suggests the Indeed parser did its job.

Another caveat is that I did not get data from any Indeed posts where the posting company used ajax or other javascript flavors to render the page (e.g., Apple). I tried for a few hours but ultimately concluded it would take me many more hours/days to figure it out (without using selenium, which I wasn't using for the Indeed scraper), and I'd rather spend my time on other things. I almost abandoned the LinkedIn scraper altogether when I realized they use this ajax business to make life hard for scrapers, but I then discovered the selenium library and continued crawling.

top10_indeed_template = jinja2.Template(
"""
// Based on http://bl.ocks.org/mbostock/3885304

require(["d3"], function(d3) {
    var data = []

    {% for row in data %}
    data.push({ 'skill': '{{ row[1] }}', 'proportion': {{ row[2] }} });
    {% endfor %}

    d3.select("#top10_indeed_d3 svg").remove()

    var margin = {top: 50, right: 50, bottom: 50, left: 10},
        width = 500 - margin.left - margin.right,
        height = 600 - margin.top - margin.bottom;

    var x = d3.scale.linear()
        .range([0, width]);

    var y = d3.scale.ordinal()
        .rangeRoundBands([0, height], .25);

    var xAxis = d3.svg.axis()
        .scale(x)
        .orient("bottom")
        .ticks(6);

    var yAxis = d3.svg.axis()
        .scale(y)
        .tickSize(0)
        .orient("right");
        
    var svg = d3.select("#top10_indeed_d3").append("svg")
        .attr("width", width + margin.left + margin.right)
        .attr("height", height + margin.top + margin.bottom)
        .append("g")
        .attr("transform", "translate(" + margin.left + "," + margin.top + ")");

    y.domain(data.map(function(d) { return d.skill; }));
    x.domain([0, d3.max(data, function(d) { return d.proportion; })]);

    svg.append("g")
        .attr("class", "xaxis")
        .call(xAxis)
        .attr("transform", "translate(" + margin.left + "," + height + ")")

    svg.append("text")
        .attr("x", (width / 2))             
        .attr("y", height + margin.bottom - 2)
        .attr("text-anchor", "middle")  
        .style("font-size", "18px") 
        .text("Proportion");

    svg.append("text")
        .attr("x", (width / 2))             
        .attr("y", 0 - (margin.top / 2))
        .attr("text-anchor", "middle")  
        .style("font-size", "18px") 
        .style("text-decoration", "underline")  
        .text("Indeed Top 10 Skills");

    svg.selectAll(".bar")
        .data(data)
        .enter().append("rect")
        .attr("class", "bar")
        .attr("x", function(d) { return margin.left; })
        .attr("width", function(d) { return x(d.proportion); })
        .attr("y", function(d) { return y(d.skill); })
        .attr("height", y.rangeBand());

    svg.append("g")
        .attr("class", "yaxis")
        .call(yAxis)
        .attr("transform", "translate(" + margin.left + ", 0)")
        .append("text")
        .style("font-size", "10px");

});
"""
)

require(["widgets/js/widget"], function(WidgetManager){
    var MultipleSelectView = IPython.DOMWidgetView.extend({
        initialize: function(parameters) {
            this.model.on('change',this.update,this);
            this.options = parameters.options;
            this.child_views = [];
            // I had to override DOMWidgetView's initialize to set model.views otherwise
            // multiple views would get attached to the model
            this.model.views = [this];
        },
        
        render : function(){
            this.$el
                .addClass('widget-hbox');
            this.$label = $('<div />')
                .appendTo(this.$el)
                .addClass('widget-hlabel')
                .hide();
            this.$listbox = $('<select/>')
                .addClass('widget-listbox')
                .attr('multiple', '')
                .attr('size', 6)
                .appendTo(this.$el);
            this.$el_to_style = this.$listbox;
            this.update();
        },
        
        update : function(options){
            if (typeof(options) === 'undefined' || options.updated_view != this) {
                var values = this.model.get('values');
                var values_order = this.model.get('values_order');
                
                var that = this;
                _.each(values_order, function(key, index) {
                    if (that.$listbox.find('option[key="' + key + '"]').length === 0) {
                        $('<option />')
                            .text(values[key])
                            .attr('key', key)
                            .appendTo(that.$listbox)
                            .on('click', $.proxy(that.handle_click, that));
                    }                    
                });
                
                var value = this.model.get('value') || [];
                
                this.$listbox.find('option').each(function(index, element) {
                    var key = $(element).attr('key');
                    
                    if (key in values) {
                        if (value.indexOf(key) != -1) {
                            $(element).prop('selected', true);
                        }
                    } else {
                        $(element).remove();
                    }
                });

                var description = this.model.get('description');
                if (description.length === 0) {
                    this.$label.hide();
                } else {
                    this.$label.text(description);
                    this.$label.show();
                }
            }
            return MultipleSelectView.__super__.update.apply(this);
        },

        handle_click: function (event) {
            var value = $(event.target).parent().children('option:selected').map(function() { return $(this).attr('key') }).get()
            
            this.model.set('value', value, {updated_view: this});
            this.touch();
        },    
    });
    WidgetManager.register_widget_view('MultipleSelectView', MultipleSelectView);
});

"""
// Based on http://bl.ocks.org/mbostock/3885304

require(["d3"], function(d3) {
    var data = []

    {% for row in data %}
    data.push({ 'skill': '{{ row[1] }}', 'proportion': {{ row[2] }} });
    {% endfor %}

    d3.select("#iInteract_d3 svg").remove()

    var margin = {top: 20, right: 20, bottom: 30, left: 50},
        width = 1100 - margin.left - margin.right,
        height = 400 - margin.top - margin.bottom;

    var x = d3.scale.ordinal()

        .scale(y)

        .attr("width", width + margin.left + margin.right)
        .attr("height", height + margin.top + margin.bottom)
        .append("g")
        .attr("transform", "translate(" + margin.left + "," + margin.top + ")");

    x.domain(data.map(function(d) { return d.skill; }));
    y.domain([0, d3.max(data, function(d) { return d.proportion; })]);

    svg.append("g")
        .attr("class", "x axis")
        .attr("transform", "translate(0," + height + ")")
        .call(xAxis);

    svg.append("g")
        .attr("class", "y axis")
        .call(yAxis)
        .append("text")
        .attr("transform", "rotate(-90)")
        .attr("y", 7)
        .attr("dy", ".71em")
        .style("text-anchor", "end")
        .text("Proportion");

    svg.selectAll(".bar")
        .data(data)
        .enter().append("rect")
        .attr("class", "bar")
        .attr("x", function(d) { return x(d.skill); })
        .attr("width", x.rangeBand())
        .attr("y", function(d) { return y(d.proportion); })
        .attr("height", function(d) { return height - y(d.proportion); });
});
"""
    )

    rendered_template = iInteract_template.render(

bubble_template = jinja2.Template(
"""
// Based on http://bl.ocks.org/mbostock/3885304

require(["d3"], function(d3) {
var diameter = 900, //max size of the bubbles
    color    = d3.scale.category20b(); //color category

var bubble = d3.layout.pack()
    .sort(null)
    .size([diameter, diameter])
    .padding(1.5);

var svg = d3.select("#bubble_d3").append("svg")
    .attr("width", diameter)
    .attr("height", diameter)
    .attr("class", "bubble");

d3.csv("ip.csv", function(error, data){

    //convert numerical values from strings to numbers
    data = data.map(function(d){ d.value = +d["proportion"]*100; return d; });

    //bubbles needs very specific format, convert data to this.
    var nodes = bubble.nodes({children:data}).filter(function(d) { return !d.children; });

    //setup the chart
    var bubbles = svg.append("g")
        .attr("transform", "translate(0,0)")
        .selectAll(".bubble")
        .data(nodes)
        .enter();

    //create the bubbles
    bubbles.append("circle")
        .attr("r", function(d){ return d.r; })
        .attr("cx", function(d){ return d.x; })
        .attr("cy", function(d){ return d.y; })
        .style("fill", function(d) { return color(d.value); });

    //format the text for each bubble
    bubbles.append("text")
        .attr("x", function(d){ return d.x; })
        .attr("y", function(d){ return d.y + 5; })
        .attr("text-anchor", "middle")
        .text(function(d){ return d["skill"]; })
        .style({
            "fill":"white", 
            "font-family":"Helvetica Neue, Helvetica, Arial, san-serif",
            "font-size": "18px"
        });
})
})"""
)
